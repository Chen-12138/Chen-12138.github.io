# webpack 的作用是什么

  • 模块打包。将不同模块的文件打包整合在一起，保证它们之间的引用正确，这样我们在开发的时候可以根据我们自己的业务划分文件模块，保证项目结构的清晰和可读性。
  • 编译兼容。有 JS 兼容，CSS 兼容等等，比如说 babel 帮我们将 js 代码转成浏览器能识别的版本，一些 css 的 loader 可以让我们使用 css 预处理器，就是让我们在开发时可以使用新特性和新语法开发，提高开发效率。
  • 能力拓展。比如说一些 plugin，比如说压缩代码，分析打包依赖，打包时间等等。

# 什么是 bundle、chunk、module

  • bundle 是 webpack 的打包产物
  • chunk 是指 webpack 在打包时分割出来的代码块
  • module 是开发的模块

# webpack 的打包流程

  • 读取 webpack 的配置参数
  • 启动 webpack，创建 Compiler 实例并开始解析项目
  • 从入口文件（entry）开始解析，并且找到其导入的依赖模块，递归遍历分析，形成依赖关系树
  • 对不同文件类型的文件用对应 loader 进行编译，loader 最终转换成 js 文件
  • 整个 webpack 打包过程还会抛出一些钩子，而 plugin 可以监听这些钩子去执行一些操作，然后改变打包产物。

文件的解析和构建是一个比较复杂的过程，webpack 时通过 Compiler 和 complication 两个核心对象实现。

Compiler 实例是一个全局单例，负责把控整个 webpack 打包流程。complication 是一个上下文对象，包含当次构建的所有信息，每次热更新和重新构建，compiler 都会重新生成一个新的 compilation 对象，负责此次更新的构建过程。

# loader 和 plugin 的区别

Loader 负责文件转换，那么 Plugin 便是负责功能扩展。

loader 遵循单一原则，只需要关心 loader 的输入和输出，传入一个 js 字符串，返回一个 js 字符串，前一个 loader 的返回值会是下一个 loader 的输入值

# tree shaking

可以将没用到的代码删除，减少打包产物大小，从而减少页面加载时间。

tree shaking 在 webpack5 中的 production 模式下自动开启，

但是注意 tree shaking 依赖于 ES Module 的静态分析，如果你的 tree shaking 没有生效，可以去找一下可以转换代码模块化方式的地方，比如说 ts，babel

# 模块热更新（HMR）

HMR 是 webpack 的一个功能，可以提高我们的开发效率，可以时代码修改后不用刷新浏览器就可以更新内容（还可以保存状态，不用重新点点点）。

原理

  • 其实是 WDS 跟浏览器会用 websocket 建立连接
  • webpack 监听文件变化，文件变化触发重新编译
    • 编译会生成新的 hash 值，改动的文件
    • 编译完后通过 socket 向浏览器推送当前 hash 值
  • 浏览器监听到文件改动推送过来新的 hash，和上次对比
    • 一致则走缓存
    • 不一致则去请求新的资源
  • 进行局部更新

# 外部拓展 Externals

防止将某些 import 的包(package)打包到 bundle 中，而是在运行时(runtime)再去从外部获取这些扩展依赖(external dependencies)

这是一个性能优化点，通过 CDN 的形式运行时引入包，而不将其打包到 bundle 中，从而减少包的体积，从而加快页面加载
